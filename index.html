Naming Conventions

Rules for naming files and paths

  • Use only lowercase letters
  • Never use spaces...ever
  • Only include letters, numbers, and dashes

Commit messages

Commit messages are a marker for what the commit will do. They should be kept brief but at least three words long and starting with an active verb. Think of the sentance as starting with "this commit will" and then begin the mssage.

HTML Elements

HTML or HyperText Markup Language is a coding language used to describe content of a website. Despite adding a difference in appearance to the content the role of HTML tags is to give content meaning and hierarchy, and should not be used to style visuals.

Commenting

Comments can be placed in code to to set reminders, explanations, and note things to collaborators. Comments are not displayed by the browser and can only be seen when reading or editing the code of a site.

Quotes and Citations

  • The quote tag is used to mark up quotes embedded in other elements like a paragraph.
  • The blockquote tag is used for large stand alone quotes.
  • The cite tag is used for marking the source of the quote. Brightspace Web Design 1 Week 4 slides

Phrasing Elements

  • The emphasis tag is used to do just that emphasize.
  • Lots of emphasis, strong importance or urgency, then we use a strong tag.
  • The small tag is used to shrink the importance of content. Is often used in footers and for copywrites.
  • Like in typography italics are used for another language, a technical term, or a title. This is visual only italics.
  • The b tag is used to define a keyword

Links

Link Text
To link text The opening a tells where the link should start and the closing /a indicates where the link ends. To create a hyperlink, you should use the a tag and href attribute, the value of which is the URL, or location, where the link is pointing to.
Link to Element on the Same Page
To link to an element on the same page an id must be used in the place of the href in the link and written starting with a hashtag.
Link to Absolute url or path
A relative file path points to a file relative to the current page.
Link to Relative url or path
Absolute URLs offer a full address for a website or other resource on the internet.
Images
Images in websites are not embedded in the website, but linked to by the HTML document. Images need two attributes to specify extra details about the image the location of the image and the alternative content.

Targeting HTML Elements

Generic Elements

There may be times when you just need a generic element to target content; most often to later manipulate styling with CSS or behavior with JS.

  • Block element used to group elements together.
  • An inline container used to mark up a part of a text, or part of a document.
  • The figure element represents self-contained content. The figcaption element represents a caption or legend describing the rest of the contents of its parent element: figure.

Common Global Attributes

Common global attributes allow us to target any HTML element within our CSS or JavaScript.

  • You can use a class multiple times throughout your HTML and attach it to different types of elements.
  • Only allowed to use one unique id once in each HTML page.

Structing Content

Header
The header is the introductory content usually at the top of the page and would include a logo or company name and main navigation. Headers are also within sections and articles to introduce the content for headings, subheadings, author, etc..
Nav
The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.
Main
The main element wraps around the main content of the page. It’s only used once per page and it tells the browser that this is where the main content is.
Section
Section is used to wrap around related pieces of content. Usually each section has its own heading.
Article
The article element represents a self-contained composition in a web page, which is intended to be independently distributable or reusable. Examples include blog post, product card, interactive widget, etc...
Footer
Many webpages end their pages with a footer at the bottom. This often includes lists of links, copyright information, extra information about the company,terms and services.
Aside
The aside element represents a portion of a document whose content is only indirectly related to the document's main content. This content will frequently be presented as a sidebar on a website.

Accessibility

Web content must be:

  • Perceivable
  • Operable
  • Understandable
  • Robust

Landmark roles

ARIA has a set of landmark roles to help those using screen readers to jump directly to specific sections within our sites.

Banner
This role is for defining an element as the masthead, added to the primary header.
Navigation
This role is for defining an element as the primary navigation, added the the top nav.
Main
This role is for defining an element as being the main content of the page, added to the main.
Complimentary
This role is for defining an element as being content related to the primary content, usually added to an aside.
Contentinfo
This role is for defining an element as information about the content: e.g. copyright, terms, privacy policy, usually added to a footer.

Aria Label

Adding aria-label attribute provides the user with a non-visual label that will only be announced by screen readers.

Image Descriptions

Always describe your images with the alt attribute. If purely decorative you can leave it empty and apply role=“presentation”

The alt attribute has two purposes:

  • To describe the photo if it doesn’t download
  • To describe the photo to someone who cannot see the image, maybe someone using a screen reader

Skip Links

Skip links are placed at the top of your page and offer users a way to jump to specific areas of the page.

Cascading Style Sheets

Commenting

Like comments in HTML, CSS comments are not seen in the browser and can only be found in the code. They are used to make notes add context and directions.

Fonts and Text

Font Properties

font-family
For setting the type of font used. Generic is included so that it can be universal to all systems regardless of available fonts.
Ex: "comic sans", sans-serif; georgia, serif; ext...
font-size
For adjusting the size of the font: a single number with a unit (rem [the best choice], em, px)
Ex: 2rem, 1.125em, 24px
font-weight
Sets how thick or thin characters in text should be displayed
Ex: bold, normal, 700
font-style
Specifies font style for text
Ex: italic or normal
line-height
For adjusting the line-height: a single number, a multiplier that multiplies the font-size
Ex: 1.5

Text Decorations

text-transform
Change capitalization such as uppercase, lowercase, capitalize. It’s a good idea to use CSS to make all caps instead of writing it directly in HTML for accessibility reasons.
Ex: uppercase, lowercase, capitalize
text-decoration
Specifies the decoration added to text. The only things in a website that should be underlined are links.
Ex: none, underline, line-through, overline
text-align
Specifies the horizontal alignment of text in an element
Ex: left, center (American spelling), right, justify
text-indent
Indent the first line of text like traditionally printed materials
Ex: 1rem, 2em, 20px, 25%
text-shadow
Adding a shadow to the text. Specified like this: text-shadow: horizontal-offset vertical-offset blur-radius colour
text-shadow: 2px 2px #ff0000;

States of Links

Links can change styles depending on the state of use.

link:
For styling the default link state.
visited:
For styling the link after it’s been visited by the user.
hover:
For styling the link when the mouse is sitting over it.
focus:
For styling the link when the keyboard has tab-focused it.
active:
For styling the link when the mouse button is pressed down on it.

Google Fonts

How to link a Google Font:

  1. Go to Google Fonts website and find a font
  2. Click on the plus icon to add the font to your selection
  3. Copy the provided HTML link
  4. Paste link in head of HTML
  5. Reference font name when applying font-family, remember to provide a back up

The Box

Margin

Outside the box is the margin, a transparent layer pushing other boxes away. Margin is the space around an element that separates it from other elements.

The margin is an invisible space around your box. It adds spacing outside of the box, pushing other boxes away from the box. Margins can have positive or negative values.

Padding

Padding adds spacing inside the box, pushing the content away from the border. The background color is visible with the padding.
